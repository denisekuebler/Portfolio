Food Use

Project made in the Module "Entwicklungsprojekt Interaktive Systeme" at the TH Köln during the Winter-Semester 2018/2019

Project Overview

  • Project from the Module "Entwicklungsprojekt Interaktive Systeme" at the TH Köln
  • Add fruit and vegetables to your inventory via QR-Code
  • The app calculates expiration dates based on factors like the kind, storage and origin country. User is notified when a food item is about to expire.
  • The app detects the ideal storage for food items based on ethylene production and stock.
  • Users can offer food that they don't want to use anymore to other users
  • The app recommends food offers that match the users personal perferences, restrictions, mobility and location.

Research

  • Researching the project domain "Food Waste in private households".
  • Identifying the problem that has to be solved.
  • Market analysis and identification of unique selling points.
  • Identification of possible users, documented through User Profiles. Work Environment Analyis and documentation of the context of use.
  • Definition of Task Scenarios and identification of user tasks.
  • Assembly of a Task Organization Model.
  • Definition of usability goals.

Early Design

  • Collecting Ideas for possible Interface solutions
  • Creating Wireframes presenting solutions
  • Evaluation of different design solutions under consideration of users requirements, restrictions and feasibility.

Detailed UI-Design

  • Creating detailed UI-Designs based on the chosen wireframes using Figma.
  • Creating a Styleguide, documenting color schemes, typography and elements based on Google's Material Design.
  • Creating an interactive Prototype using Figma. This one was used to evaluate the screens with potential users.
  • Test-users were given different tasks and were asked to complete those. The evaluation was done using the method Think Aloud.
  • Problems which occured were noted and the interface was adjusted to reduce the problems.

My Role in the Project

  • The project was done in groups of 2, the requirement was to hand in a finished app and a documentation containing the work process.
  • My job was doing everything regarding the field of UI-UX Design.
  • This included everything documented in in this portfolio.
